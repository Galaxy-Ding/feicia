# 验收计划

## 1. 验收目标

确认改造完成后，系统可以稳定支撑“导演统一创意判断、服化道锁定视觉基准、分镜师严格适配 Seedance 2.0”的目标流程。

## 2. 验收阶段

### 阶段一 架构验收

- schema 文件存在
- validator 存在
- asset registry 存在
- reference map 存在

### 阶段二 功能验收

- 三阶段命令可运行
- 真实图片可登记
- Prompt 只引用 READY 资产
- 审核失败可回修

### 阶段三 质量验收

- 导演负例可被拒收
- Seedance 约束负例可被拒收
- 平均分与单项阈值可被正确执行

### 阶段四 交付验收

- 文档完整
- 测试完整
- 样例产物可追溯

## 3. 验收前置条件

- 所有 P0 任务完成
- 自动化测试通过
- 至少 1 套完整样例从剧本走到最终 Prompt
- 至少 1 套变体资产样例完成验证

## 4. 验收角色

- 产品负责人：确认流程符合方法论
- 技术负责人：确认架构与规则可维护
- 业务审核人：确认讲戏、服化道、Prompt 质量
- 合规审核人：确认规则覆盖平台红线

## 5. 验收输出

- 验收记录
- 问题关闭清单
- 最终验收报告

## 6. 不通过条件

- 仍然只能靠 Skill 文本维持质量
- 无法判断单项分是否低于 6
- Storyboard 仍可在无真实参考图时继续
- 会话连续性仍未接入调用
- 关键负例无法被拒收

## 7. 当前复验进度

### 已满足

- schema / validator / asset registry / reference-map 已存在
- 三阶段命令可运行
- 真实图片可登记
- Prompt 阶段在无真实图片资产时会阻断
- 审核失败可回修
- 平均分与单项阈值已可结构化执行
- 会话连续性已接入调用

### 仍待补齐

- 黄金样例验收
- 变体资产样例验收
- 最终人工验收记录与关闭清单
