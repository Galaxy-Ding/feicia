# 需求报告

## 1. 背景

本项目目标不是普通的 Prompt 生成器，而是一套围绕 Seedance 2.0 的影视化内容生产流水线。核心目标是把“导演脑海中的画面”稳定翻译成下游可执行的技术提示词，并通过两步审核保证业务质量和平台合规。

## 2. 现状评估

当前工程满足“流程存在”，不满足“质量硬控制”。

已满足：

- 三阶段流程存在。
- 七类 Skill 已覆盖导演、服化道、分镜和审核方法论。
- CLI、日志、报告、阶段接受状态完整。

未满足：

- 缺少导演讲戏质量的结构化规则校验。
- 缺少服化道全局资产库的实体化管理。
- 缺少 Seedance 引用和节拍约束的程序化验收。
- 缺少结构化评分协议与自动拒收规则。
- 缺少真实参考图资产到 `@引用` 的追踪链路。

## 3. 目标需求

### 3.1 业务需求

- 导演输出必须以具体物理动作驱动画面，禁止空泛抽象词替代动作。
- 每个剧情点必须形成完整动作链和镜头链。
- 运镜必须有方向、起点和终点。
- 光影必须具备光源、方向、色温、强度或明暗关系。
- 下游分镜师只需要忠实转写导演讲戏，不承担创意判断。
- 服化道需要跨集累积人物与场景资产，支持“新增 / 复用 / 变体”。
- 每个阶段都必须执行业务审核和合规审核，且反馈一次性合并给对应 Agent。
- Seedance 提示词必须遵守素材引用上限、节拍密度、安全区和镜头可执行性。

### 3.2 工程需求

- 审核协议必须结构化，不能只依赖自然语言报告。
- 所有核心约束必须能被程序检查，而不是只写进 Skill。
- 资产状态必须可追踪，至少包含“提示词已生成 / 图片已产出 / 图片已登记 / 可引用”。
- 会话连续性必须真实参与后续生成与修订。
- 测试必须覆盖正向与负向质量场景。

## 4. 功能需求

### FR-01 导演讲戏结构化输出

- 导演产物除 Markdown 成文外，还要生成结构化元数据。
- 每个剧情点至少包含：剧情点 ID、角色、场景、镜头组类型、建议时长、动作节拍数、安全区说明、导演讲戏正文。

### FR-02 导演讲戏规则校验

- 校验是否出现抽象概念替代动作。
- 校验是否存在动作跳步。
- 校验镜头运动是否有方向性。
- 校验光影描述是否达到最小具体度。
- 校验时长与节拍密度是否匹配。

### FR-03 服化道资产注册

- 建立角色与场景的全局资产注册表。
- 每个资产必须有稳定 ID，不再仅依赖“ep01 block”。
- 资产必须区分提示词版本和图片版本。

### FR-04 参考图落地链路

- 支持记录实际生成出的角色图、场景宫格图、场景拆图。
- 支持登记本地路径、来源工具、版本、创建时间和状态。
- Storyboard 阶段必须引用已登记的真实图片资产，而不是仅引用提示词文本。

### FR-05 Seedance Prompt 硬校验

- 校验每条 Prompt 的引用数量。
- 校验引用是否都已在资产映射表登记。
- 校验场景宫格拆图是否为独立场景引用。
- 校验节拍密度和安全区。
- 校验镜头运动与动作链的可执行性。

### FR-06 审核协议升级

- 业务审核和合规审核必须返回结构化 JSON。
- 业务审核 JSON 至少包含：维度分项分数、平均分、是否存在单项低于 6、结论、问题列表、修订建议。
- 合规审核 JSON 至少包含：规则命中项、风险级别、结论、问题列表、修订建议。

### FR-07 自动接受规则

- 只有当业务审核平均分大于等于 8、所有单项大于等于 6、合规审核 PASS 时，阶段才可自动 ACCEPT。
- 任一条件不满足，必须进入待人工复核或待修订。

### FR-08 会话连续性

- 后续生成、审核、修订必须可携带同一 episode 同一角色历史上下文。
- 会话不能只落盘，不参与模型调用。

### FR-09 测试与追踪

- 每个阶段必须保留输入、输出、审核摘要、修订轮次、版本关联。
- 测试必须包含失败样例与边界样例。

## 5. 非功能需求

- 可追溯：任一 Prompt 可追踪到导演剧情点、资产 ID、审核轮次。
- 可维护：规则应数据化，避免写死在多个 Prompt 文本中。
- 可扩展：未来可新增平台合规规则或新增分镜平台。
- 可审计：所有接受与拒绝结论必须有结构化证据。

## 6. 约束条件

- 参考图生成本身仍可保持人工执行。
- 工程当前以本地 CLI 运行为主，不强制引入 Web UI。
- 保持现有三阶段主干不被推翻，优先增量改造。
