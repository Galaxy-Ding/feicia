# 需求报告

## 1. 项目名称

即梦 2.0 浏览器自动化工作流项目

## 2. 项目背景

当前团队需要把“图片生成与短视频生成”从人工点击流程升级为可重复执行的自动化流水线。目标平台为即梦 2.0 网页端，执行框架为 OpenClaw，自定义 workflow 负责串联浏览器操作、本地文件处理、评分决策和日志沉淀。

用户已经明确选择的路线为：

- 人工首次登录即梦网页
- OpenClaw 接管后续网页点击与输入
- 自动下载与规范命名图片
- 将图片交给另一个评分角色独立打分
- 对合格图片保留前后帧
- 使用合格帧生成 0-5 秒短视频

## 3. 业务目标

1. 把图片和视频生成流程从“人盯页面操作”改造成“系统自动执行，人工例外处理”
2. 降低高频重复动作的人工成本，包括找输入框、等待完成、下载、改名、筛图
3. 提高结果可追踪性，使每个产物都能追溯到提示词、生成批次、评分和后续用途
4. 为后续多集、多镜头批量生产建立统一工作流基础

## 4. 目标用户

- AI 分镜师团队的执行人员
- 负责批量出图、筛图、出短视频的制作成员
- 后续需要接手问题定位、质量复盘和验收的技术负责人

## 5. 核心使用场景

### 场景 A：单镜头出图

系统根据任务单打开即梦图片生成页，输入提示词，发起生成，等待图片完成，下载到本地并命名。

### 场景 B：多候选图评分筛选

系统将下载结果提交给评分角色，根据评分阈值自动决定保留、淘汰或重试。

### 场景 C：前后帧驱动短视频

系统从评分合格的图片中选择前后帧，进入即梦视频生成页，完成 0-5 秒视频任务提交、等待和下载。

### 场景 D：异常中断恢复

遇到登录失效、验证码、页面改版、下载失败、评分失败时，系统记录问题并暂停到可恢复节点。

## 6. 功能需求

### FR-01 登录态复用

- 系统支持使用人工首次登录后的浏览器 profile
- 系统在任务启动前检查登录态是否可用
- 登录失效时暂停任务并提示人工重新登录

### FR-02 页面导航与元素定位

- 系统自动进入即梦图片生成页
- 系统识别提示词输入框、生成按钮、任务列表、下载入口
- 系统支持主选择器和后备选择器

### FR-03 图片生成执行

- 系统读取任务提示词
- 系统自动输入提示词并发起生成
- 系统等待页面出现“生成完成”或可下载状态

### FR-04 图片下载与命名

- 系统将图片下载到本地 staging 目录
- 系统按照统一命名规则重命名并归档
- 系统保留原始下载文件名与规范文件名映射

### FR-05 图片评分流转

- 系统将图片路径、任务信息、提示词交给评分角色
- 评分角色输出结构化结果，包括总分、分项分、评语、是否合格
- 系统根据规则自动筛选通过与失败图片

### FR-06 前后帧保留规则

- 系统为每个镜头保留通过评分的前帧和后帧
- 缺失任一关键帧时不得进入视频生成阶段
- 支持人工标记某一张图为前帧或后帧

### FR-07 视频生成执行

- 系统打开即梦视频生成页
- 系统上传或引用前后帧
- 系统提交 0-5 秒视频生成任务
- 系统等待视频完成并下载到本地

### FR-08 日志与证据

- 系统记录每次页面跳转、输入、点击、下载、评分、筛选、视频生成
- 系统输出任务级日志、错误日志和验收证据
- 系统支持按任务 ID 查询整个执行链路

### FR-09 失败重试与人工接管

- 支持配置自动重试次数和间隔
- 超过重试阈值后任务转人工处理
- 支持从失败节点恢复，而不是整条链路重跑

## 7. 非功能需求

### NFR-01 稳定性

- 对同一页面改版要具备一定容错能力
- 对网络波动、慢加载、下载延迟有超时与重试策略

### NFR-02 安全性

- 不在任务配置或日志中明文保存账号密码
- 对 cookie、profile、下载内容目录做最小权限管理

### NFR-03 可维护性

- workflow、页面定位规则、命名规则、评分阈值应配置化
- 浏览器步骤与本地后处理逻辑解耦

### NFR-04 可审计性

- 每个产物都要能追溯到任务、提示词、评分和下载记录
- 每次失败都要有明确错误原因与发生阶段

## 8. 约束条件

- 即梦网页可能存在验证码、风控、限流、DOM 改版
- 登录需要人工首次处理，且登录态存在过期风险
- 不同浏览器、分辨率、语言环境可能影响元素定位
- 视频生成实际参数受即梦页面能力限制，0-5 秒为业务目标，实施时需要与页面当前能力对齐

## 9. 成功标准

1. 在连续 10 个图片任务中，自动成功率不低于 80%
2. 图片下载后命名正确率达到 100%
3. 评分结果能够结构化落盘，且筛选逻辑可复验
4. 至少完成 3 个“前后帧合格后自动生成视频”的闭环样例
5. 任一失败任务都能在日志中定位到失败步骤和错误原因
