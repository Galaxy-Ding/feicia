# 项目范围

## 1. 本期范围内

- 建立 `docs/character-action/` 文档包
- 定义小说人物角色系统的统一实施边界
- 设计中文与英文两条处理链路
- 设计 LangGraph 状态图、节点职责与中断恢复点
- 设计 JSON Schema 契约与版本策略
- 设计 SQLite / Postgres 数据模型
- 设计测试、验收、问题与任务推进机制

## 2. 本期范围外

- 不在本阶段直接开发完整后端代码
- 不在本阶段接入完整前端审核台
- 不在本阶段完成所有 LLM prompt 的最终定稿
- 不在本阶段完成向下游分镜系统的真实 API 联调
- 不在本阶段完成大规模语料评测与线上压测

## 3. 系统边界

### 3.1 本系统负责

- 小说文本导入与章节化
- NLP 预处理
- 人物发现、归并、关系抽取
- 证据回链
- QA 校验
- 审核回写
- 结构化入库

### 3.2 本系统暂不负责

- 分镜镜头生成
- 视觉角色设定图生成
- 资产检索与素材下载
- TTS、配音、渲染、成片

## 4. 实施阶段范围

### Phase 01

- 文档与契约基线
- 架构和数据模型冻结

### Phase 02

- 核心代码骨架
- 基础抽取链路
- SQLite 本地落地

### Phase 03

- 人工审核闭环
- Postgres 生产化
- 稳定 QA 与回放能力

### Phase 04

- 下游系统联调
- 批处理与性能优化
- 监控告警与运维补齐

## 5. 交付物

- 方案文档包
- 统一状态图设计
- 统一 Schema 设计
- 统一数据库设计
- 任务清单
- 测试计划
- 验收计划与清单

## 6. 本期完成定义

只有满足以下条件，本期文档阶段才算完成：

- 文档包完整
- 需求、范围、架构、详细设计已闭环
- 测试计划和验收计划已明确
- 任务清单和问题日志已建立
