# 场景设计技能

[技能说明]
    场景设计技能，将小说章节中的环境描写、动作发生地和情绪氛围整理为可直接使用的场景提示词。

    核心目标有两个：
    1. 从章节中提取真正关键的场景节点
    2. 结合风格设定，输出统一、可执行的场景提示词

    输出重点不是剧情复述，而是空间、时间、光影、道具、氛围和整体视觉风格。

[文件结构]
    secene-design-skill/
    ├── SKILL.md                                  # 本文件（技能流程说明）
    └── templates/
        └── secene-design-template.md             # 场景设计产出模板

[执行流程]

    第一步：读取输入信息
        - 读取当前章节的小说正文
        - 读取用户提供的风格设定或参考风格描述
        - 读取已有的 assets/scene-prompts.md（如存在），保持场景体系一致

    第二步：拆解关键场景
        从章节中提取需要做视觉设定的场景，优先保留：
        - 发生关键剧情的主要空间
        - 情绪反差明显的转场空间
        - 视觉辨识度高、后续会复用的场景

        一个章节中同一地点如果只有昼夜、天气、破损程度等状态变化，可标为同场景变体。

    第三步：整理场景信息
        为每个场景提炼以下维度：
        - 场景名
        - 地点类型
        - 时间
        - 空间布局
        - 关键物件/道具
        - 光源与光线方向
        - 色调和材质
        - 氛围情绪

        如果原文环境描写有限：
        - 基于剧情功能做保守补全
        - 补全必须服从世界观、时代背景和风格要求

    第四步：融合风格内容
        把用户给定的风格信息融合到每个场景提示词中，例如：
        - 写实电影感
        - 国漫厚涂
        - 日漫清透光感
        - 蒸汽朋克、赛博都市、东方奇幻等世界观风格

        风格融合要求：
        - 是“统一视觉语言”，不是简单在结尾补一句风格名
        - 同一章节的场景应保持同一世界观和美术逻辑

    第五步：编写场景提示词
        每个场景提示词必须包含：
        - 空间描述
        - 光线描述
        - 主要景别或观察视角
        - 关键道具和材质细节
        - 氛围和情绪
        - 风格表达

        写法要求：
        - 使用叙事描述式段落
        - 不要关键词堆叠
        - 不要把剧情过程整段搬进提示词
        - 重点保证画面可视化和可生成

    第六步：判断素材状态
        对每个场景标注状态：
        - 新增：第一次出现的场景
        - 复用：与已有场景基本一致
        - 变体：同一场景在时间、天气、破损、陈设、氛围上出现明显变化

    第七步：输出
        按照 templates/secene-design-template.md 的格式输出。

        推荐写入：
        - assets/scene-prompts.md

        写入方式：
        - 首章创建文件
        - 后续章节按章节追加
        - 只为新增和变体项写完整提示词

[写作原则]
    - 先写空间，再写光线，再写氛围和风格
    - 场景要能让读者一眼看出“这是哪里、什么时候、什么气氛”
    - 场景提示词服务于画面生成，因此细节要具体但不要散乱
    - 同一章内的提示词要保持整体视觉一致

[注意事项]
    - 如果小说中的场景只是功能性转场、没有独立视觉价值，可不单独建条目
    - 如果同一地点多次出现，优先建立基础设定，再把后续变化标为变体
    - 风格内容必须真正融入画面描述，不能只停留在标签层面
