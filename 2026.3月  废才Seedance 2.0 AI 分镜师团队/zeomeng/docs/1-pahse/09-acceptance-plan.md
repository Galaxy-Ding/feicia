# 验收计划

## 1. 验收目标

确认一期已经达到当前机器边界下的交付标准，并为后续真实浏览器联调保留明确的二次验收步骤。

## 2. 验收范围

- 登录态复用接口
- 图片生成页定位接口
- 提示词自动提交接口
- 图片生成等待接口
- 图片下载与重命名
- 日志、状态、映射记录

## 3. 验收方式

- 文档验收
- 本地模拟运行验收
- 证据文件验收
- 后续真实环境现场演示验收

## 4. 分层验收步骤

### 4.1 当前机器验收

1. 检查 `workflow/` 配置模板与提示词样例
2. 运行 `python run_pipeline.py run --browser mock`
3. 检查是否生成 `downloads/staging/` 原始文件
4. 检查是否生成 `downloads/images/` 归档文件
5. 检查是否输出映射 JSONL、运行日志、任务状态文件
6. 运行 `pytest`
7. 检查测试结果是否全部通过

### 4.2 后续真实环境验收

1. 人工确认账号已登录
2. 由 OpenClaw / browser skill 接管浏览器
3. 检查是否进入即梦图片生成页
4. 检查提示词是否成功提交
5. 检查是否等待到生成完成
6. 检查所有图片是否下载成功
7. 检查是否完成规范命名和映射记录
8. 检查日志、错误记录与截图证据
