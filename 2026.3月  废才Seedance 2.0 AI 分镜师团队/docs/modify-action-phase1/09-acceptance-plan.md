# 验收计划

## 1. 验收目标

确认第一阶段已经形成后续可持续推进的统一工程骨架。

## 2. 验收范围

- 文档完整性
- 统一代码工作区
- 统一 CLI 骨架
- 路径与契约能力
- 自动化测试
- 阶段总结与问题闭环

## 3. 验收阶段

### 阶段一 文档验收

- 文档包完整
- 内容与现有子系统边界一致

### 阶段二 代码验收

- `video_only_once_phase1/` 目录存在
- 可以解析统一工作区
- 可以输出桥接命令

### 阶段三 测试验收

- 单元、功能、集成、安全测试通过

### 阶段四 管理验收

- 阶段总结已输出
- 阶段问题点已输出
- 问题日志已更新

## 4. 不通过条件

- 仍没有统一代码工作区
- 仍没有统一文档目录
- 无法识别两个子系统路径
- 测试未通过
- 阶段总结和问题日志未形成闭环

## 5. 验收输出物

- 验收清单
- 测试结果
- 阶段总结
- 最终验收报告
