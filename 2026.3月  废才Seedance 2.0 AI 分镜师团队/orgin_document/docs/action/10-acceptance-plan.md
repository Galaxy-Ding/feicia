# 验收计划

## 1. 验收目标

确认复刻项目是否已经达到“可按原方法稳定生产分镜文件”的交付标准。

## 2. 验收对象

- 编排引擎
- Prompt 资产接入方式
- 模型配置能力
- 文件输出结构
- 三阶段执行能力
- 审核回修能力

## 3. 验收角色

| 角色 | 职责 |
|---|---|
| 甲方/业务负责人 | 确认结果是否符合原方法 |
| 工程负责人 | 演示运行过程 |
| Prompt 负责人 | 解释 Prompt 资产映射 |
| 测试负责人 | 出示测试证据 |

## 4. 验收前提

- 文档完整
- 测试完成
- 至少 1 个单集完整样例
- 至少 1 个跨集样例
- 问题日志中无阻断项

## 5. 验收步骤

1. 检查目录结构与配置文件
2. 检查 Prompt 资产是否正确接入
3. 演示 `~status`
4. 演示 `~start`
5. 演示阶段一审核闭环
6. 演示 `~design`
7. 演示阶段二审核闭环
8. 演示 `~prompt`
9. 演示阶段三审核闭环
10. 演示跨集复用
11. 演示修改请求回流
12. 检查最终产物结构与内容

## 6. 验收证据

需要保留以下证据：

- 输入剧本文件
- 输出文件
- 审核日志
- 运行日志
- 测试记录
- 问题关闭记录

## 7. 验收通过条件

同时满足以下条件：

1. 三阶段完整跑通
2. 审核失败可自动回修
3. 跨集复用正常
4. 输出结构与原项目定义一致
5. 所有 P0 问题关闭

## 8. 验收不通过条件

任一项出现则不能通过：

- 无法稳定生成标准文件
- 审核闭环缺失
- Agent 恢复机制失效
- 素材映射不符合 Seedance 约束
- 输出内容结构与模板偏离严重
