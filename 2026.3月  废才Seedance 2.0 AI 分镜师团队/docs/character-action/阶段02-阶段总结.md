# 阶段02-阶段总结

## 1. 阶段目标

把 `docs/character-action/` 的方案文档，落成可运行的最小工程骨架，为后续真实 NLP、抽取、审核和编排打基础。

## 2. 开始前回顾

本阶段开始前，已回顾 `阶段01-阶段总结.md`，并按其中三条高优先级结论推进：

- 先建立统一中间结构
- 先固化 evidence span 规则
- 先把工程边界从分镜系统中拆开

## 3. 本阶段完成情况

已完成：

- 新建独立工程目录 `character_action/`
- 建立 `prepare` / `preprocess-book` / `status` CLI
- 建立 YAML 配置与运行目录初始化
- 建立 `BookRecord`、`ChapterRecord`、`ChunkRecord`、`MentionRecord`、`EvidenceRecord`、`NormalizedTextUnit`
- 建立 chapter split、chunk 切分、evidence span 编码规则
- 建立中文 `HanLP` fallback adapter 与英文 `BookNLP` fallback adapter
- 建立 SQLite repo、复合主键约束和运行态统计
- 建立 JSONL 运行日志
- 准备中英文样书并完成功能验证
- 完成单元、功能、集成、安全测试

未完成：

- 真实 `HanLP` 与 `BookNLP` 引擎接入
- alias merge
- relation build
- review queue
- LangGraph 状态图与 checkpoint
- Postgres
- `video_only_once_phase1` 对 `character_action` 的正式桥接

## 4. 本阶段核心结论

- `character_action` 应独立于 `fenjing_program`，不能把小说角色系统直接塞进分镜工程
- 统一调度入口仍应保留在 `video_only_once_phase1`
- 章节、chunk、mention、evidence 的命名规则是书内唯一，数据库必须以 `book_id + object_id` 做复合约束
- 在真实 NLP 引擎接入前，先用 fallback adapter 固化 contract 是可行路径
