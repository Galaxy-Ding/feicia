# 需求报告

## 1. 项目名称

即梦 2.0 浏览器自动化项目一期

## 2. 项目背景

当前团队希望先把“即梦网页图片生成”中的高频重复动作自动化。为了控制实施风险，一期只做最小闭环，不处理评分、筛选和视频阶段。

本期目标是让 OpenClaw 在人工首次登录后，稳定接管浏览器，自动完成图片页定位、提示词输入、生成等待、图片下载和规范命名。

补充说明：

- 一期技术选型明确为 `OpenClaw + browser skill + 本地文件处理脚本`
- 现阶段不在当前电脑部署 OpenClaw
- 因此本期文档中的 OpenClaw 是“目标执行框架定义”，不是“当前机器立即安装要求”
- 当前机器可以先承担文档设计、配置准备、目录规范和后处理脚本设计，不承担 OpenClaw 运行时部署

## 3. 一期业务目标

1. 降低人工重复点击成本
2. 打通“提示词到本地图片文件”的自动化闭环
3. 建立稳定的登录态复用、下载落盘和日志追踪基础
4. 为二期评分与视频能力预留扩展接口

## 4. 目标用户

- 即梦图片制作执行人员
- 需要批量出图的分镜或美术团队成员
- 后续需要接手维护 workflow 的技术人员

## 5. 核心使用场景

### 场景 A：人工登录后自动出图

人工先登录即梦网页并保存浏览器状态，系统随后自动进入图片生成页，输入提示词并生成图片。

### 场景 B：批量提示词执行

人工提前准备提示词列表，系统逐条提交并下载全部图片到指定目录。

### 场景 C：失败后恢复

如果页面超时、下载失败或登录失效，系统记录到失败节点，便于人工处理后重启。

## 6. 功能需求

### FR-01 人工登录与登录态保存

- 系统支持人工首次登录即梦网页
- 系统支持后续启动时复用已保存的浏览器 profile 或登录态
- 系统在任务开始前检查登录态是否仍有效

### FR-02 图片生成入口定位

- 系统自动进入即梦图片生成页面或图片生成栏位
- 系统支持主选择器和备用选择器
- 页面定位失败时给出明确错误信息

### FR-03 提示词输入与提交

- 系统从外部任务文件读取提示词
- 系统自动输入提示词
- 系统自动点击生成按钮
- 支持批量提示词逐条执行

### FR-04 生成等待

- 系统自动等待图片生成完成
- 系统识别图片已可下载的状态
- 超时后支持有限次数重试

### FR-05 图片下载与重命名

- 系统自动下载所有生成图片到指定目录
- 系统保留原始下载文件名
- 系统根据命名规则生成规范文件名
- 系统输出原始文件名与新文件名映射

### FR-06 日志与执行记录

- 系统记录任务开始、页面定位、提交生成、等待、下载、命名等关键步骤
- 失败时记录原因、截图和阶段

## 7. 非功能需求

### NFR-01 稳定性

- 页面加载慢时有超时与重试机制
- 页面局部结构变化时有选择器后备方案

### NFR-02 安全性

- 不在代码或日志中明文保存账号密码
- 登录态文件只用于本地执行环境

### NFR-03 可维护性

- 选择器、下载目录、命名规则、超时阈值应配置化

### NFR-04 可追踪性

- 每个下载文件都可追溯到提示词、执行时间和任务 ID

## 8. 约束条件

- 人工必须先完成登录
- 即梦网页可能存在验证码、风控和 DOM 改版
- 浏览器下载权限必须在本地环境中提前放开
- 当前这台电脑不部署 OpenClaw，本期只保留与 OpenClaw browser skill 兼容的接口和配置设计

## 9. 成功标准

1. 连续 10 次图片任务中，自动成功提交率不低于 80%
2. 下载成功的图片命名正确率达到 100%
3. 任一失败任务都能在日志中定位失败阶段
4. 可对同一批提示词执行批量下载并完成文件归档
