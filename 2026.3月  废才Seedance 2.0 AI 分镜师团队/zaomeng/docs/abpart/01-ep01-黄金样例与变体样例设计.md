# ep01 黄金样例与变体样例设计

## 1. 文档目标

这份文档解决两件事：

- 帮 `ep01` 设计“最值得补的黄金样例”和“最值得补的变体样例”
- 把这次设计过程拆开记录，方便后续复刻成类似 agent

## 2. 这次我是怎么做的

### 2.1 打算怎么做

我的思路不是先凭感觉点图，而是先回答三个问题：

1. `ep01` 的核心叙事重心是谁
2. `ep01` 的关键线索场景是什么
3. 哪些图最能证明这套流程已经具备“角色资产 + 场景资产 + 变体资产”的闭环能力

基于这三个问题，我采用了下面的筛选顺序：

1. 先找主角色
2. 再找关键线索角色
3. 再找关键线索场景
4. 再找能天然做变体的一组角色或场景

### 2.2 用到了什么工具

这次只用了本地终端读取文件，没有调用外部网络：

- `Get-ChildItem`
  - 用来确认 `docs/`、`assets/`、`outputs/` 当前是否存在真实验收材料
- `Get-Content`
  - 读取 `script/ep01-时辰守门局.md`
  - 读取 `assets/character-prompts.md`
  - 读取 `assets/scene-prompts.md`
- 额外参考了本项目已有代码逻辑：
  - `src/feicai_seedance/acceptance_runner.py`
  - `README.md`

### 2.3 遇到什么困难

这次有三个真实困难：

- 困难 1：当前仓库里没有 `outputs/` 目录，也没有已经登记好的真实图片资产
  - 结果：我没法从“已有成图”里挑样例，只能从剧本和提示词设计里反推“应该补哪些图”
- 困难 2：终端里部分中文显示有编码噪声
  - 结果：我不能完全依赖终端显示的中文细节，只能结合结构、角色名、段落关系和上下文交叉判断
- 困难 3：当前代码只检查样例目录里“是否存在图片”，还不会自动判断这些图和 registry 的语义绑定关系
  - 结果：这次设计必须额外强调“优先放那些已经或准备登记进 registry 的图”，否则验收会通过，但证据价值偏弱

## 3. 我读到的 ep01 关键信息

从 `script/ep01-时辰守门局.md` 和资产 prompt 可归纳出：

- 主角色是“顾星澜”
- 关键被救对象是“沈钧白”
- 核心对立角色是“萧景琰”
- 关键线索场景是“沈钧白书房”
- 关键推演场景是“顾星澜算房”
- 故事张力强的场景还包括：
  - 司天台
  - 太史局书房外廊
  - 沈府门前
  - 殿阁回忆场景

所以，样例不该平均铺开，而要优先证明：

- 主角色能稳定生成
- 关键线索场景能稳定生成
- 同一角色 / 同一场景可以做出可辨认的变体

## 4. 黄金样例怎么选

### 4.1 选择原则

黄金样例是“这集最标准、最能代表最终交付水准”的图。

对 `ep01` 来说，最好的黄金样例不应该是边缘人物，而应该覆盖：

- 主角色标准形象
- 关键剧情锚点角色
- 关键线索场景
- 关键推演场景

### 4.2 建议的最小黄金样例集

如果你只想先补最小可交付集合，我建议放 3 张：

1. 顾星澜标准角色设定图
2. 沈钧白标准角色设定图
3. 沈钧白书房场景图

### 4.3 建议的完整黄金样例集

如果你想让最终验收更扎实，我建议放 4 张：

| 优先级 | 类型 | 建议内容 | 推荐文件名 | 为什么选它 |
|---|---|---|---|---|
| P0 | 角色黄金样例 | 顾星澜标准角色设定图 | `char-gu-xinglan-golden-standard.png` | 她是主角，后续很多判断都围绕她展开 |
| P0 | 角色黄金样例 | 沈钧白标准角色设定图 | `char-shen-junbai-golden-standard.png` | 他是事件起点和情感锚点，决定整集成立与否 |
| P0 | 场景黄金样例 | 沈钧白书房线索场景图 | `scene-shen-study-golden-clue.png` | 这是核心取证场景，最能体现“线索场景”的交付价值 |
| P1 | 场景黄金样例 | 顾星澜算房推演场景图 | `scene-gu-room-golden-strategy.png` | 这是主角推演与布局的空间，能证明“角色外的剧情空间”也稳定成立 |

## 5. 变体样例怎么选

### 5.1 选择原则

变体样例不是“再补几张别的图”，而是要证明：

- 同一角色在不同状态下仍可辨认
- 同一场景在不同叙事状态下仍可辨认

最有价值的变体，不是完全无关的新图，而是围绕黄金样例做变化。

### 5.2 建议的最小变体样例集

如果你只想先补最小可交付集合，我建议放 2 张：

1. 顾星澜角色变体图
2. 沈钧白书房场景变体图

### 5.3 建议的完整变体样例集

| 优先级 | 类型 | 建议内容 | 推荐文件名 | 变体维度 | 为什么选它 |
|---|---|---|---|---|---|
| P0 | 角色变体样例 | 顾星澜夜间观测 / 推演状态变体图 | `char-gu-xinglan-variant-night-duty.png` | 同角色，不同状态或光线 | 主角必须先证明可做稳定变体 |
| P0 | 场景变体样例 | 沈钧白书房“搜查后”变体图 | `scene-shen-study-variant-searched.png` | 同场景，不同叙事状态 | 最能体现“线索被翻动后”的变化价值 |
| P1 | 场景变体样例 | 顾星澜算房“深夜烛光 / 拂晓冷光”变体图 | `scene-gu-room-variant-dawn-shift.png` | 同场景，不同时段光线 | 很适合证明环境资产的可复用性 |
| P1 | 角色变体样例 | 萧景琰权力压迫态变体图 | `char-xiao-jingyan-variant-court-pressure.png` | 同角色，不同场合气场 | 可作为高权力角色的补充证据，但优先级低于主角和核心场景 |

## 6. 最推荐的落地顺序

为了最小成本通过并且让证据有意义，我建议按这个顺序补图：

1. 先补 3 张黄金样例
   - 顾星澜标准图
   - 沈钧白标准图
   - 沈钧白书房图
2. 再补 2 张变体样例
   - 顾星澜变体图
   - 沈钧白书房变体图
3. 如果还有余力，再补
   - 顾星澜算房黄金图
   - 顾星澜算房变体图

## 8. 给后续 agent 的可复刻规则

如果后面要做成 agent，我建议把“设计样例”的规则固定成下面 6 步：

1. 读取剧本
2. 读取角色 prompt
3. 读取场景 prompt
4. 识别主角色、关键线索角色、关键线索场景、关键推演场景
5. 先产出最小样例集，再产出完整样例集
6. 输出建议文件名、目录路径和补图优先级

## 9. 当前结论

对 `ep01` 而言，最值得优先补的不是“平均挑几张图”，而是围绕下面 5 个锚点补图：

- 顾星澜标准角色
- 沈钧白标准角色
- 沈钧白书房黄金场景
- 顾星澜角色变体
- 沈钧白书房场景变体

这 5 张图，是我认为当前性价比最高、最适合先把最终验收证据补起来的一组。
