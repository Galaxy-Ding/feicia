# 项目范围

## 1. 实施目标

本项目的范围是把当前 Prompt 工程方法复刻成一套可运行的本地工程，不包含视频平台本身，也不包含真实图片/视频生成平台的二次开发。

## 2. 范围内

### 2.1 流程编排

- 制片人总控编排
- 导演 Agent
- 服化道 Agent
- 分镜师 Agent
- 三阶段流转
- 双审核闭环
- 同集恢复、跨集重置

### 2.2 文件系统与产物管理

- `script/` 剧本输入
- `assets/` 素材提示词沉淀
- `outputs/` 分集产物输出
- `.agent-state.json` 状态恢复
- 日志文件

### 2.3 模型接入

- OpenAI 兼容接口接入
- 角色级模型配置
- 重试、超时、限流、失败日志

### 2.4 Prompt 资产接入

- 原项目 `CLAUDE.md`
- 原项目 `agents/*.md`
- 原项目 `skills/**/*.md`
- 原项目 `templates/*.md`

### 2.5 验证与验收

- 测试计划
- 验收计划
- 验收清单
- 最终验收报告模板

## 3. 范围外

以下内容不在本期范围内：

- 文生图平台的自动出图执行
- Seedance 平台自动上传素材与自动出片
- 九宫格场景图的自动切图工具
- GUI 可视化后台
- 团队协作权限系统
- 财务、计费、套餐管理
- 多租户 SaaS 化

说明：原项目在阶段二到阶段三之间，默认存在一个人工步骤，即用户用人物/场景提示词去出图，并将场景宫格切成单独参考图。该步骤与原方法一致，保留为人工边界，不强行自动化。

## 4. 交付边界

本次交付的目标是“可以稳定生成文件”，不是“自动完成整条影视生产链”。

最终至少应交付：

- 可运行的流程引擎
- 可配置的模型接入层
- 可复用的 Prompt 资产目录
- 标准化的输出目录
- 测试与验收文档

## 5. 假设前提

- 用户已具备可调用的大模型 API。
- 用户能提供规范剧本文件。
- 用户可接受中间人工出图与素材整理步骤。
- 用户接受第一阶段以本地文件系统作为持久化方式。

## 6. 不可妥协项

以下能力必须保留，否则不视为“原方法复刻”：

1. 三阶段拆分
2. 导演在阶段一执行、在全阶段审核
3. 审核必须是“业务审核 + 合规审核”
4. `outputs/<集数>/01-director-analysis.md`
5. `assets/character-prompts.md`
6. `assets/scene-prompts.md`
7. `outputs/<集数>/02-seedance-prompts.md`
8. `.agent-state.json` 恢复机制
