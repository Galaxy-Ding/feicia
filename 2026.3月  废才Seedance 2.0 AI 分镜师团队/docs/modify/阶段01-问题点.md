# 阶段 01 问题点

## 1. 已发现但未彻底解决的问题

- `DIRECTOR` 结构化转换目前基于 Markdown 解析，尚未做到模型直接输出原生 JSON，后续仍有解析脆弱性。
- `DESIGN` 阶段当前只有基础结构校验，还没有形成角色 / 场景 / 变体的正式 schema。
- `SEEDANCE` 校验目前已覆盖引用数量和音频设计缺失，但尚未覆盖节拍密度、安全区、九宫格拆图独立性。
- 资产 registry 目前只完成文件初始化，还没有生命周期写入、图片登记和 reference-map 构建。
- `SessionStore` 历史仍未真正回注到模型调用，这个高优先级问题尚未处理。
- `status.py` 仍未纳入 `IMAGE_PENDING`、`REFERENCE_MAPPING_PENDING` 等状态。
- `llm_client.py` 的明文 API Key 回退仍是安全风险，当前阶段未处理。

## 2. 建议处理方案

- Phase 2：优先加强 validator，减少 Markdown 解析路径中的误判和漏判。
- Phase 3：实现 registry 的写入、查询、复用、变体和图片登记。
- Phase 4：让 Pipeline 直接消费 registry 和 reference-map，而不是继续直接吃 Markdown 全文。
- 安全项：为 API Key 策略增加显式开关，默认禁止明文 key。

## 3. 需要用户决策时的触发条件

- 如果要把导演阶段改成“模型一次直接返回 Markdown + JSON 双产物”，需要确认是否允许调整现有 Prompt 模板。
- 如果要禁用明文 API Key 回退，需要确认当前运行环境是否已经具备环境变量注入方式。
- 如果要把 design 阶段从 Markdown 资产文本切换为 registry-first，需要确认是否允许调整当前人工出图流程。
