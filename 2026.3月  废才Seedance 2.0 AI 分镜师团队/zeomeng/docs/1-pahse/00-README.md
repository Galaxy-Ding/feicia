# 即梦一期浏览器自动化文档包

## 1. 文档目的

本目录用于沉淀“即梦 2.0 网页自动化”一期范围内的实施文档，并记录当前机器已经完成的本地落地内容。

一期目标仍然限定为：

1. 人工登录系统，完成登录并保存登录状态
2. 自动找到图片生成页面或图片生成栏位
3. 自动输入人工预先准备的提示词，并点击生成按钮
4. 自动等待图片生成完成，将所有图片下载到指定目录并重新命名

一期仍不包含评分、筛选、前后帧保留和视频生成。

## 2. 当前实施状态

截至 `2026-03-25`，当前机器已经完成以下本地可实施内容：

- 建立 `src/zaomeng_automation/` 一期代码骨架
- 落地任务编排、状态文件、运行日志、错误日志、下载映射记录
- 落地 `BrowserOperator` 抽象接口与 `MockBrowserOperator` 模拟实现
- 跑通“提示词读取 -> 模拟提交 -> 模拟下载 -> 重命名 -> 映射记录”闭环
- 补齐单元、功能、集成、安全测试，并通过 `pytest`
- 新增 `docs/phase-reports/`，记录 6 个阶段的总结与问题点

当前机器仍未做真实 OpenClaw / browser skill 联调。

## 4. 当前关键原则

- 一期技术选型明确为 `OpenClaw + browser skill + 本地文件处理脚本`
- 当前机器不部署 OpenClaw，只负责本地脚本层、接口边界、配置模板、测试和文档
- 登录必须由人工完成，自动化流程只复用登录态
- 下载文件必须先原样保留，再做规范命名，确保可追溯
- 所有敏感字段不得以明文写入运行日志
- 每进入新阶段前，需要回顾上一阶段总结；本次已通过阶段文档固化

## 5. 当前完成定义

从“本机实施边界”角度，一期当前可视为完成以下交付：

1. 已完成后续对接 OpenClaw 的接口边界与本地可运行骨架
2. 已完成提示词、选择器、目录、日志、命名和映射规则模板
3. 已完成模拟浏览器环境下的一期流程闭环验证
4. 已完成自动化测试与阶段文档追踪机制
5. 已明确真实浏览器联调与最终验收的待办项
