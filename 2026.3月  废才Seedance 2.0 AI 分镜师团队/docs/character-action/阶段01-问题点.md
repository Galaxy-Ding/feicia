# 阶段01-问题点

## 1. 高优先级

- 必须先定义 `HanLP` 与 `BookNLP` 的统一 adapter 输出，否则后续抽取器无法复用
- 必须先定义 evidence span 编码规则，否则人工审核和关系回链会失真
- 必须把 alias merge 设计成可审核、可回写的机制，不能只依赖一次性自动归并

## 2. 中优先级

- 需要尽快准备中英文样书和人工标注基线
- 需要尽快锁定 LLM provider、模型档位和成本预算
- 需要尽快确认 HanLP 模型许可和生产部署边界

## 3. 低优先级

- 前端审核台可以后置
- 下游消费 API 可以在 SQLite 验证通过后再补

## 4. 处理原则

- 高优先级问题不解决，不进入大规模实现
- 中优先级问题应在 Phase 02 中段前解决
- 所有问题都要同步更新到 `07-issue-log.md`
