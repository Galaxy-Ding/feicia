# 阶段03-问题点

## 1. 高优先级

- relation build 已有初版，但当前仍以 deterministic 局部共现规则为主，复杂隐式关系还没有进入图
- alias merge 已有初版，但准确率风险仍在，必须继续扩大 review queue 覆盖范围
- native 依赖虽然已在本机跑通，但安装条件较苛刻，必须尽快固化成脚本或 CI 约束

## 2. 中优先级

- 需要补更多中英文样书，建立真实抽取基线
- 需要把 review task 继续扩展到关系冲突、属性冲突和更细粒度 relation normalize
- 需要补 Postgres 与 SQLite 的并行验证
- `BookNLP` 英文链路当前依赖 `HF_ENDPOINT=https://hf-mirror.com` 才能稳定下载 Hugging Face 模型，后续需要明确正式环境镜像策略

## 3. 低优先级

- `doctor` 目前只检查模块可导入性，后续可以补 spaCy 模型、BookNLP 模型文件与 Hugging Face 资源检查
- exports 当前以 JSON 为主，Markdown 报告可以后续补

## 4. 处理原则

- 下一阶段开始前，必须先回顾本文件与 `阶段03-阶段总结.md`
- 如果真实依赖安装失败或模型资源异常，再按规则去查官方文档或开源实现
- 新增问题继续同步更新 `07-issue-log.md`
