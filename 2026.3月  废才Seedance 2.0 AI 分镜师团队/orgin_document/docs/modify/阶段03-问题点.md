# 阶段 03 问题点

## 1. 当前仍未解决的问题

- registry 当前更偏向“文件级登记”，还没有形成完整的复用 / 变体决策引擎。
- `scene_panel` 当前需要人工逐张登记图片，还没有自动拆图流程。
- `reference-map.json` 已可构建，但还没有和 Prompt 校验的所有技术约束全量联动。
- 设计阶段仍未输出正式 JSON sidecar。

## 2. 建议处理方案

- 下一阶段优先让 session、状态机和引用校验与 registry 真正联动。
- 后续应考虑让 design 直接产出结构化 JSON，而不是继续依赖 Markdown 解析。
- 如要处理自动拆图，建议优先搜索开源图像裁切与 panel 检测实现，再决定是否引入依赖。

## 3. 可能需要用户决策的点

- 是否允许后续引入自动拆图库或图像处理依赖。
- 是否允许把 design 阶段升级为 Markdown + JSON 双产物。
