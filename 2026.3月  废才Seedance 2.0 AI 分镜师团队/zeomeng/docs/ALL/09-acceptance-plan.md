# 验收计划

## 1. 验收目标

确认项目已达到“可执行、可恢复、可追踪、可交接”的交付标准。

## 2. 验收角色

- 业务验收人：确认图片和视频结果满足生产需要
- 技术验收人：确认自动化链路、日志和恢复机制可用
- 执行验收人：现场跑通样例并记录证据

## 3. 验收范围

- 浏览器 profile 复用
- 图片生成链路
- 下载与命名链路
- 评分与筛选链路
- 视频生成链路
- 异常中断与人工接管
- 日志、报告和证据

## 4. 验收方式

### 4.1 文档验收

- 检查需求、架构、设计、测试、问题日志是否齐备

### 4.2 演示验收

- 现场跑通至少 1 个完整图片到视频闭环任务

### 4.3 证据验收

- 核对日志、截图、下载文件、评分报告、状态文件

## 5. 验收前提

- 测试账号可用
- 浏览器 profile 已准备
- 下载目录可写
- 样例任务已准备
- 问题日志中高优先级问题已处理

## 6. 验收步骤

1. 运行登录态校验
2. 运行图片生成任务
3. 检查图片下载、命名和元数据
4. 执行评分并检查筛选结果
5. 检查前后帧是否齐备
6. 运行视频生成任务
7. 下载视频并检查落盘
8. 审阅日志、问题记录和证据

## 7. 验收输出物

- 验收执行记录
- 验收截图
- 样例图片与视频
- 评分报告
- 最终验收报告
