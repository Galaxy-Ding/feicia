# 阶段 02 问题点

## 1. 当前仍未解决的问题

- 节拍密度和安全区规则目前是启发式实现，还没有基于显式镜头时长和 beat 列表做严格判定。
- 设计阶段仍未输出正式结构化 JSON，宫格校验现在仍依赖 Markdown 解析。
- Prompt 阶段虽然能拦截整图宫格误用，但还没有真正和 `reference-map.json`、registry 图片状态联动。
- registry 目前只有初始化，没有资产增删改查、复用、变体和图片登记逻辑。
- Session continuity、status 扩展和 API Key 安全策略仍未处理。

## 2. 建议处理方案

- Phase 3 先把 registry 与 reference-map 做实，再让 validator 使用真实资产元数据。
- Phase 4 再处理 session history 回注和状态机升级，避免中途反复改 Pipeline 输入。
- 对安全区和节拍密度，后续应优先让导演 / Prompt 结构化产物直接输出时长与 beats，而不是继续从自由文本估算。

## 3. 可能需要用户决策的点

- 是否允许在 design 阶段新增独立 JSON sidecar，替代只靠 Markdown 解析。
- 是否允许把 storyboard 入口改成 registry-first / reference-map-first。
- 是否允许禁用明文 API Key 回退并改成环境变量强依赖。
