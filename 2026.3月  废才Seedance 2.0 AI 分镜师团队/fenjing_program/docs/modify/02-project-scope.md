# 项目范围

## 1. 改造目标

本次改造目标是把当前“提示词驱动流程”升级为“规则驱动的内容生产系统”，让高质量要求从 Skill 文本上升为程序能力。

## 2. 范围内

- 导演讲戏结构化 schema 设计与校验。
- 服化道资产库的 ID 化、状态化、跨集复用机制。
- 真实参考图登记与 `@引用` 映射机制。
- Seedance Prompt 静态校验器与审核 JSON 协议。
- 审核结论自动判定逻辑升级。
- 会话历史真实接入模型调用。
- 测试体系补强。
- 文档、目录、状态检测逻辑更新。

## 3. 范围外

- 文生图工具本身的自动调用与自动出图。
- Seedance 平台的真实视频生成自动回写。
- 图像质量的 CV 自动识别。
- Web 前端产品化界面。
- 多人协作权限系统。

## 4. 交付物

- 升级后的代码实现。
- 新的数据目录与资产索引文件。
- 新的结构化报告格式。
- 完整测试用例。
- 更新后的使用文档与验收文档。

## 5. 阶段划分

### 阶段 A 规则建模

- 建 schema
- 建 validator
- 建审核 JSON 协议

### 阶段 B 资产闭环

- 建资产注册表
- 建真实参考图登记机制
- 建引用映射

### 阶段 C 流水线接入

- Pipeline 接入 schema、validator、review gate、history

### 阶段 D 测试与验收

- 单元测试
- 集成测试
- 黄金样例验收

## 6. 成功标准

- 不是“能生成文件”，而是“能拒收低质量文件”。
- 不是“有审核报告”，而是“审核结论能被机器稳定执行”。
- 不是“有资产提示词”，而是“Storyboard 能引用真实资产”。
