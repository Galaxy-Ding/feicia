# 阶段03-阶段总结

## 1. 阶段目标

在 Phase 02 最小骨架之上，补齐抽取与审核的第一轮可执行能力，并把上游角色系统正式接入统一调度入口。

## 2. 开始前回顾

本阶段开始前，已回顾 `阶段02-阶段总结.md` 与 `阶段02-问题点.md`，按其中优先级推进：

- 先补真实预处理器接入路径
- 再补 alias merge 和 review queue
- 再把 `character_action` 纳入 `video_only_once_phase1`

## 3. 本阶段完成情况

已完成：

- 为 `HanLP` / `BookNLP` 增加 native-or-fallback 双路径
- 新增 `doctor` 命令，输出本机依赖可用性
- 新增运行时 schema validator
- 新增 alias merge engine
- 新增 review queue builder
- 新增 `extract-characters` 命令
- 新增 `build-relations` 命令与 `relation_graph.json` 导出
- 新增 `run-qa` 命令与 `qa_findings.json` 导出
- 扩展 SQLite：`characters`、`review_tasks`
- 扩展 SQLite：`relations`
- 扩展 SQLite：`qa_findings`
- 扩展 `status` 输出角色数、关系数、QA finding 数与审核任务数
- 在 `2026-03-28` 本机完成 `HanLP` / `BookNLP` native 依赖安装与样书实测
- 扩展 `video_only_once_phase1`：
  - 新增 `character_action` workspace 路径
  - 新增上游 manifest 字段
  - 新增桥接命令
- 新增 mock native 测试与端到端集成测试

未完成：

- LangGraph
- checkpoint/recovery
- Postgres
- 更丰富的 QA / review task 类型

## 4. 本阶段核心结论

- “真实引擎接入”不仅已接入口，而且已在 `2026-03-28` 本机完成中英文样书实测
- alias merge 不能只做自动聚合，必须与 review queue 一起落地
- `video_only_once_phase1` 现在已经能把 `character_action` 纳入统一清单，不再是口头约定
- native 运行不能依赖“最新包默认可用”，必须锁定 `setuptools`、`torch`、`transformers` 版本，并为英文链路准备 Hugging Face 镜像方案
- relation build 已可在真实样书上产出第一版关系图，但当前仍主要覆盖局部显式关系，需要后续继续补 QA、冲突检测与复杂关系类型
- QA rules 已开始把代词角色名、别名冲突、低证据人物、低置信度关系显式拉入审核队列，但关系冲突和属性冲突仍需继续扩展
