# 阶段01-阶段总结

## 1. 阶段目标

建立“小说人物角色系统”的方案级实施基线，确保后续开发不再围绕抽象方向反复讨论，而是直接进入可执行的工程拆解。

## 2. 本阶段完成情况

已完成：

- 统一文档目录建立
- 双语技术路线明确
- LangGraph 编排层设计明确
- JSON Schema 与数据库模型明确
- 任务、测试、验收与问题基线建立

未完成：

- 代码骨架
- 样书测试
- 审核系统实现
- 下游联调

## 3. 本阶段核心结论

- 中文与英文小说不应各自独立建系统，必须共用统一上层状态与数据模型
- `HanLP` 与 `BookNLP` 只能做预处理入口，不能取代角色系统主编排
- `LangGraph` 适合作为长流程、可恢复、可人工介入的主干
- 最终数据一定要经过 Schema 校验后才能入库或下游消费

## 5. 下一阶段建议

- 先做最小可运行链路，不要一开始追求全量字段
- 优先攻克统一中间结构、alias merge、evidence store
- 用 SQLite 跑通样书后再切 Postgres
