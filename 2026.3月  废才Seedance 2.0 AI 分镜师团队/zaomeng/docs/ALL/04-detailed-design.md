# 详细设计

## 1. 工作流概览

完整流程分为 7 个步骤：

1. 校验浏览器 profile 和登录态
2. 打开即梦图片生成页并提交提示词
3. 等待图片生成完成并下载到本地
4. 重命名、归档并写入图片元数据
5. 交给评分角色打分并执行筛选
6. 为合格任务保留前后帧
7. 打开即梦视频生成页并提交 0-5 秒视频任务

## 2. 角色与责任

### 2.1 调度角色

- 输入任务参数
- 驱动各阶段执行
- 更新任务状态

### 2.2 浏览器执行角色

- 负责图片页和视频页的所有网页动作
- 不负责业务判断

### 2.3 评分角色

- 独立读取图片与上下文
- 输出结构化评分
- 不直接改写源图片文件

### 2.4 人工介入角色

- 处理验证码、风控、DOM 重大变更、命名争议和验收裁决

## 3. 图片生成步骤设计

### 3.1 登录态校验

- 启动浏览器 profile
- 打开即梦首页或目标页
- 判断是否存在用户头像、创作入口或其他登录后元素
- 若检测失败，则标记 `BLOCKED` 并提示人工重登

### 3.2 页面定位

建议配置以下元素定位策略：

- 提示词输入框：主定位使用输入区域标签或 placeholder，后备定位使用文本相邻区域和可编辑区域
- 生成按钮：主定位使用按钮文本，后备定位使用按钮 role 和页面区域相对位置
- 历史任务卡片：主定位使用任务列表容器，后备定位使用最新生成区域
- 下载按钮：主定位使用下载文本或图标，后备定位使用悬停菜单

### 3.3 提交任务

- 读取任务提示词
- 清空输入框旧内容
- 输入提示词
- 可选填写宽高比、风格、数量等参数
- 点击生成按钮
- 写入 `submitted_at`

### 3.4 等待完成

完成判定至少满足其一：

- 页面出现可点击下载按钮
- 任务卡片状态从“生成中”变为“完成”
- 缩略图预览加载完成且附带下载入口

等待策略建议：

- 默认轮询间隔 5 秒
- 单次最长等待 10 分钟
- 超时后重试 1 到 2 次
- 超过阈值后转人工

## 4. 下载与命名设计

### 4.1 下载目录

下载先进入 `downloads/staging/<task_id>/`

### 4.2 规范命名

建议命名格式：

`{episode}_{scene}_{shot}_{role}_{variant}_{stage}_{score}_{timestamp}.png`

示例：

`ep01_sc03_sh012_keyframe_A_v02_pass_8.7_20260325T141530.png`

字段说明：

- `episode`：集数
- `scene`：场次
- `shot`：镜头号
- `role`：`pre`、`post`、`keyframe`
- `variant`：候选版本号
- `stage`：`raw`、`pass`、`reject`
- `score`：评分总分，保留一位小数
- `timestamp`：生成完成时间

### 4.3 元数据记录

每张图都应记录：

- 任务 ID
- 原始文件名
- 规范文件名
- 提示词
- 提交时间
- 下载时间
- 页面截图路径
- 评分结果路径

## 5. 评分设计

### 5.1 输入

- 图片路径
- 提示词
- 镜头上下文
- 预期用途

### 5.2 输出结构

建议 JSON 字段：

- `task_id`
- `image_path`
- `overall_score`
- `dimension_scores`
- `pass`
- `summary`
- `issues`
- `recommended_role`

### 5.3 评分维度

- 构图
- 人物一致性
- 场景一致性
- 镜头叙事有效性
- 可作为视频前后帧的稳定性

### 5.4 通过规则

建议采用：

- 总分大于等于 8.0
- 任一单项不得低于 6.0
- `recommended_role` 必须能落到 `pre` 或 `post`

## 6. 前后帧筛选设计

### 6.1 基本规则

- 每个镜头至少保留一张前帧和一张后帧
- 如果只出现一类帧，则任务保持 `FRAME_READY_PARTIAL`
- 只有前后帧都齐备时，才能进入视频阶段

### 6.2 冲突处理

- 多张图都被判定为前帧时，优先选总分最高者
- 分数相同则优先最新批次
- 仍无法判断时转人工

## 7. 视频生成设计

### 7.1 页面准备

- 打开即梦视频生成页
- 检查上传控件或帧输入区域可用

### 7.2 输入材料

- 上传前帧
- 上传后帧
- 填写视频提示词或过渡说明
- 设置时长为目标 0-5 秒

### 7.3 结果获取

- 等待视频任务完成
- 下载到 `downloads/videos/<task_id>/`
- 写入视频元数据与关联的前后帧信息

## 8. 异常处理设计

### 8.1 必须转人工的情况

- 登录失效
- 验证码出现
- 页面结构大改导致元素定位失败
- 下载按钮消失或下载行为被浏览器拦截

### 8.2 可自动重试的情况

- 页面短时加载失败
- 元素暂未出现
- 下载超时
- 视频任务轮询超时一次

### 8.3 恢复点

建议支持从以下节点恢复：

- 图片页已打开
- 图片任务已提交
- 图片已下载未命名
- 图片已评分未筛选
- 前后帧已确定未提交视频
- 视频已提交未下载
