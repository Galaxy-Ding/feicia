4. 生成过程

4.1 这两个文件的直接生成过程

  1. 读取剧本，先跑阶段一 start ep01，生成导演分析文件。
  2. 导演阶段必须先被接受，否则 run_design() 会直接报错阻断。
  3. 阶段二读取 01-director-analysis.md，同时读取历史 character-prompts.md 和 scene-prompts.md 作为增量上下文。
  4. 系统拼接阶段二 prompt：总控 CLAUDE.md + art-designer + art-design-skill + gemini-image-prompt-guide + 人物/场景示例 + 模板。
  5. 调用模型键 art_designer 生成一整份阶段二 Markdown，内容同时包含人物提示词和场景提示词。
  6. 程序先做业务审核和合规审核，必要时会自动回修，最多 3 轮。
  7. 最终结果通过 split_art_design_artifact() 拆成两部分。
  8. 程序用 upsert_episode_block() 写回 assets/character-prompts.md 和 assets/scene-prompts.md，按集数块追加或更新。
  9. 程序再调用 sync_design_assets()，把角色条目、场景宫格条目、场景 panel 条目同步进 assets/registry/asset-registry.json。

4.2 整个项目里的完整闭环

  1. 剧本分析：start ep01 产出导演讲戏本。
  2. 服化道设计：design ep01 产出本报告关注的两个 Markdown 资产文件。
  3. 角色资产链：extract-characters → build-character-sheets → export-character-image-tasks → generate-character-images → review-character-images → approve-character-assets。
  4. 场景资产链：export-scene-image-tasks → generate-scene-images → review-scene-images → approve-scene-assets。
  5. 引用映射：build-reference-map 基于导演 JSON 和 READY 资产生成引用表。
  6. 分镜阶段：prompt ep01 消费导演分析、资产库和引用映射，生成最终 Seedance 分镜提示词。
